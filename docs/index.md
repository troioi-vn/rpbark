# Documentation Index

This directory holds lightweight system documentation for `R.P.Bark`.

The goal is not to mirror the code line by line. The goal is to explain the parts of the project that are harder to recover from code alone: gameplay intent, animation rules, art assumptions, and structure that should stay consistent as the prototype grows.

## Principles

- Keep docs short, current, and useful during active development.
- Document systems and behavior, not every single file.
- Prefer explaining `why` and `how this should feel`, not only `what exists`.
- Treat animation and art-pipeline notes as real engineering documentation.
- When a system becomes annoying to re-explain, give it its own page here.

## Docs

- [Player System](./player.md): movement, run toggle, animation states, sprite-sheet assumptions, and tuning notes for the player character.

## Notes

- `tmp/learning_curve.md` is intentionally not part of the shared project documentation set. It is a local working memory file for guided learning, not a repo-facing system doc.
